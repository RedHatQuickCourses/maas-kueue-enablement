Lab: Kueue demo - setup and verify

This walkthrough shows how to apply the demo manifests from the chapter attachments and verify the Kueue objects using the OpenShift web terminal.

This walkthrough uses two personas (Team A and Team B) to demonstrate queue admission, priority ordering, and how to interpret Kueue objects.

Personas

  • Team A — exploratory workloads, low priority, may consume large amounts of resources.

  • Team B — critical workloads, high priority, small footprint but needs fast admission.

Step 1 — Apply base demo manifests (namespaces, flavor, queue)

From the OpenShift web terminal run:

oc apply -f modules/ch5-kueue/attachments/kueue-demo/00-namespaces.yaml
oc apply -f modules/ch5-kueue/attachments/kueue-demo/05-priorityclasses.yaml
oc apply -f modules/ch5-kueue/attachments/kueue-demo/10-resourceflavor.yaml
oc apply -f modules/ch5-kueue/attachments/kueue-demo/20-clusterqueue.yaml
oc apply -f modules/ch5-kueue/attachments/kueue-demo/30-localqueues.yaml

Step 2 — Run Team A (resource hog)

# Team A submits a large background job that requests most of the ClusterQueue quota
oc apply -f modules/ch5-kueue/attachments/kueue-demo/41-team-a-hog-workload.yaml

Step 3 — Observe admission and resource consumption

oc get workload -A
oc describe workload team-a-hog -n team-a-project
oc get pods -n team-a-project
oc get clusterqueue shared-cluster-queue -o yaml

Step 4 — Now submit Team B (high priority)

oc apply -f modules/ch5-kueue/attachments/kueue-demo/42-team-b-critical-workload.yaml

Observe how Kueue orders admission. In many Kueue deployments priority (via Kubernetes PriorityClass) affects admission order for queued workloads; the higher-priority workload should be admitted before lower-priority ones when quota is available. Note: Kueue’s admission behavior and live preemption of already-admitted workloads depends on operator configuration — this lab demonstrates admission ordering, not forced eviction.

oc get workload -A
oc describe workload team-b-critical -n team-b-project
oc get pods -n team-b-project

Cleanup

To remove the demo resources:

oc delete -f modules/ch5-kueue/attachments/kueue-demo/

If any objects fail to delete you may need to manually remove finalizers or dependent CRs.
